Atlas Fund Capital — Blog MCP Server

Push blog posts directly from Claude.ai to atlasfundcapital.com
Copy-paste, plug-and-play. No coding required.

What This Does

You tell Claude "Write a blog post about SBA loans" and Claude writes it, formats it with proper headings, detects the right category, and publishes it live on your website — instantly. You only need to upload the featured image manually afterward.

What You Need

  • Your Ubuntu server where atlasfundcapital.com is hosted
  • SSH access to that server (PuTTY or Terminal)
  • 5 minutes

Step-by-Step Setup

1 SSH into your server

Open Terminal (Mac/Linux) or PuTTY (Windows) and connect:

ssh your-username@your-server-ip

2 Go to your project folder

cd /path/to/your/loan_project

Replace /path/to/your/loan_project with the actual path where your Django project lives (where manage.py is).

3 Install the required Python packages

pip install -r requirements.txt

If pip isn't found, try pip3 instead.

4 Test the MCP server starts correctly

python mcp_blog_server.py

You should see:

Atlas Blog MCP Server starting on 0.0.0.0:8001
MCP endpoint: http://0.0.0.0:8001/mcp/blog

Press Ctrl+C to stop it. If it starts without errors, you're good.

5 Set it up to run 24/7 as a systemd service

Create a service file so the MCP server auto-starts and stays running even after reboots:

sudo nano /etc/systemd/system/mcp-blog.service

Paste this in (edit the paths to match your setup):

[Unit]
Description=Atlas Fund Capital Blog MCP Server
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/var/www/loan_project
Environment=SITE_URL=https://atlasfundcapital.com
Environment=MCP_PORT=8001
ExecStart=/usr/bin/python3 /var/www/loan_project/mcp_blog_server.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target

IMPORTANT: Change /var/www/loan_project to your actual project path. Change /usr/bin/python3 to your Python path (find it with which python3). Change www-data to the user your Django app runs as.

Save and exit (Ctrl+O, Enter, Ctrl+X), then:

sudo systemctl daemon-reload
sudo systemctl enable mcp-blog
sudo systemctl start mcp-blog
sudo systemctl status mcp-blog

You should see active (running) in green.

6 Add Nginx reverse proxy

This makes https://atlasfundcapital.com/mcp/blog point to the MCP server running on port 8001.

Open your Nginx config:

sudo nano /etc/nginx/sites-available/atlasfundcapital

Add this block before the existing location / block:

# --- MCP Blog Server ---
location /mcp/blog {
    proxy_pass http://127.0.0.1:8001;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 86400s;
    proxy_send_timeout 86400s;
    proxy_buffering off;
}

The long timeouts are necessary — Claude keeps an SSE connection open. Without them, Nginx will cut the connection after 60 seconds.

Test and reload Nginx:

sudo nginx -t
sudo systemctl reload nginx

7 Verify it's publicly accessible

curl https://atlasfundcapital.com/mcp/blog

You should get a JSON response (not a 404). If you get a Django 404 page, the proxy isn't routing correctly — double-check the nginx location block is placed before the Django proxy pass.

8 Connect Claude.ai

  1. Open claude.ai in your browser
  2. Click your profile icon (bottom-left) → Settings
  3. Go to the Developer tab
  4. Click Edit Config or Manage MCP Servers
  5. Click Add Server
  6. Fill in:
    • Name: Atlas Blog
    • URL: https://atlasfundcapital.com/mcp/blog
    • Transport: Streamable HTTP
  7. Click Save

You should see a green checkmark and two tools listed: create_blog_post and list_recent_posts.

How to Use It

A Create a blog post

In any Claude conversation, just ask:

Write a blog post about how SBA 7a loans help small businesses get working capital. Format it with H1 and H2 headings, and publish it to the blog.

Claude will write the post with proper HTML formatting and call the create_blog_post tool automatically. It detects the category (e.g., "SBA Loans"), generates the excerpt, and publishes it instantly.

Claude will respond with the published URL, like:

Blog post created and published successfully!

Title: "How SBA 7a Loans Fuel Small Business Growth"
Category: SBA Loans (auto-detected)
URL: https://atlasfundcapital.com/blog/how-sba-7a-loans-fuel-small-business-growth/

B Upload the featured image

Claude can't generate images. Go to your admin panel:

https://atlasfundcapital.com/superadmin/blog/

Click Edit on the new post, upload a featured image, and save.

C See your recent posts

Ask Claude:

Show me the last 5 blog posts

Claude will call list_recent_posts and show you what's live.

Troubleshooting

Problem Fix
MCP server won't start sudo journalctl -u mcp-blog -n 50 to see the error log. Usually a wrong Python path or missing dependency.
404 from /mcp/blog The Nginx location /mcp/blog block must come before the location / block that proxies to Django. Order matters in Nginx.
Claude says "tool not found" Restart the MCP server: sudo systemctl restart mcp-blog. Then reconnect in Claude.ai settings.
Connection drops frequently Make sure proxy_read_timeout 86400s is in your Nginx config. Without it, Nginx kills the connection after 60s.
"Cannot call from async context" error Install asgiref: pip install asgiref. The latest mcp_blog_server.py already uses sync_to_async.
